# AdSense Safety Guardrails

목표: 애드센스 승인/운영 리스크를 줄이고 정책 위반 가능성을 사전에 차단한다.

## Allowed

- 광고는 콘텐츠 문맥 안에서 자연스럽게 배치한다.
- 본문 시작 전 1개, 본문 하단 1개 등 고정 위치로 운영한다.
- 광고 영역에는 `Sponsored` 등 명확한 구분 라벨을 사용한다.
- 모바일 퍼포먼스를 해치지 않는 정적/경량 UI를 유지한다.

## Forbidden

- 결과 확인 전 강제 대기 후 광고 노출
- 광고를 버튼/카드처럼 오인하게 만드는 UI
- "클릭하세요" 등 직접 클릭 유도 문구
- 본문보다 광고가 먼저 보이도록 과도한 배치
- 페이지 전환마다 과도한 광고 재초기화

## Review Checklist

1. 광고가 핵심 콘텐츠를 가리지 않는가
2. 광고와 CTA가 시각적으로 명확히 구분되는가
3. 결과/공유 흐름을 광고가 강제 중단하지 않는가
4. 인앱 브라우저에서 레이아웃 깨짐이 없는가
5. LCP/CLS 저하 없이 유지되는가

## Incident Rule

- 정책 의심 이슈가 발견되면 즉시 롤백한다.
- 롤백 후 `docs/SCALING_EXECUTION_CHECKLIST.md`의 리스크 섹션에 원인/대응을 기록한다.
